ALGORITHM

Reliable Token Generation

To prevent double-booking and queue errors, the system relies on safe, guaranteed database updates to generate token numbers perfectly in order.

queue_service.py 
async def generate_token(db, queue_id):
    queue = await db.queues.find_one_and_update(
        {"_id": queue_id},
        {"$inc": {
            "token_counter": 1, 
            "waiting_count": 1
        }},
        return_document=ReturnDocument.AFTER
    )
    return queue["token_counter"]
DATA CONTRACT

Token Booking Definition

How a token is structured and tracked from the patient's phone to the hospital server.

Frontend Ticket (React)

The digital ticket shown to the patient on their phone:

  • tokenNumber: Format like T-005
  • estimatedWaitTime: Calculated wait time
  • priority: Standard vs High Priority
  • status: Pending payment / Confirmed

Backend Record (MongoDB)

The official record saved in the hospital database:

  • queue_id: Links token to doctor's queue
  • patient_id: Identifies the registered user
  • appointment_type: Regular OPD visit
  • status: Waiting ➔ Calling ➔ Completed
SPAM PREVENTION

Geofenced Live Booking

To prevent patients from reserving slots they will not attend, standard token bookings are locked to the hospital surroundings.

Location Validation

The React client requests coordinate access, checking the patient's distance against the hospital's registered latitude and longitude.

Server-Side Enforcement

The API denies tokens if coordinates exceed the threshold radius, ensuring only physical arrivals enter the active OPD queue.

LOAD BALANCING

Smart Workload Allocation

The advanced booking engine automatically load-balances incoming appointments by checking doctor specializations and active queue workloads.

  • Filters doctors matching case specialization keywords.
  • Aggregates active booking totals across candidate profiles.
  • Assigns patient to the doctor with the lowest active load.
AT-HOME BOOKING CRITERIA

Reserved Priority Demographics

At-home advance booking is reserved for specific patient categories to avoid queue manipulation.

Pregnancy Cases

Expectant mothers can book remotely to bypass early hospital travel and crowd exposures.

Infants & Children

Reserved strictly for children below 7 years to protect them from high-risk clinic environments.

Elderly Patients

Dedicated access for seniors aged 70 and above to promote healthcare accessibility.

EMERGENCY POLICY

OPD Focus Only

NOQ strictly operates as an outpatient workflow optimizer. Critical life-threatening situations bypass the application completely.

Zero App Intervention

Emergency cases (trauma, cardiac arrests, acute distress) must bypass the digital queue for direct physical admission.

Direct Triage

The system includes warning notices: in case of critical illness, patients are directed to call emergency hotlines or visit the ER.

No VIP Skips

The system prevents jumping the line. Everyone must wait their turn fairly.

First-In, First-Out

The doctor's screen forces them to call patients in exact check-in order.
