NOQ – Smart Hospital Queue Management

Enterprise‑grade, role‑based outpatient workflow platform

Prepared for final stakeholder meeting before deployment.

Technologies: React 19, Vite, FastAPI, MongoDB, Docker‑Compose, CI/CD

Problem Statement

  • Waiting‑room congestion – patients waste hours with no visibility.
  • Unstructured token systems – paper/ verbal tokens cause VIP bias and errors.
  • Revenue leakage – manual calculations lead to accounting mismatches.
  • Missing clinical data – no digital EMR, prescriptions, or analytics.

Solution Overview

NOQ replaces the analog queue with a real‑time digital token system that:

  • Provides live token tracking on mobile/web.
  • Enforces strict FIFO ordering with zero VIP overrides.
  • Integrates payment, EMR, and automated revenue split.
  • Supports advanced booking for vulnerable groups while geofencing normal bookings.

High‑Level Architecture

Frontend

React 19 + Vite, WebSocket live updates, Geolocation API, role‑based dashboards.

Backend API

FastAPI (async ASGI), JWT‑based RBAC, background tasks for audit logging.

Data Store

MongoDB – document model for flexible clinical records, atomic $inc token counter.

All services are containerised (Docker‑Compose) and orchestrated via CI/CD pipelines.

RBAC Enforcement

def require_role(allowed: List[Role]):
    def dependency(token: str = Depends(oauth2_scheme)):
        payload = jwt.decode(token, SETTINGS.SECRET, algorithms=["HS256"])
        if payload["role"] not in allowed:
            raise HTTPException(status_code=403)
        return payload
    return Depends(dependency)

FastAPI dependencies validate JWT role before any endpoint logic runs.

Patient‑to‑Clinic Booking Lifecycle

  1. Initial Selection: Choose hospital branch and medical specialty.
  2. Smart Allocation: System automatically assigns the least‑loaded doctor & room.
  3. Secure Payment: Fee collected via integrated gateway; confirmation stored.
  4. Geofence & Token Generation: Normal bookings require the user to be within a 500 m radius; advanced bookings (pregnant, children ≤ 7, elders ≥ 70) are exempt. After validation a sequential token is issued.

Core Algorithms & Policies

Atomic Queue Counter

async def generate_token(db, queue_id):
    result = await db.queues.find_one_and_update(
        {"_id": queue_id},
        {"$inc": {"counter": 1}},
        return_document=ReturnDocument.AFTER,
    )
    return result["counter"]

Geofence Validation (Haversine)

function distance(lat1, lon1, lat2, lon2) {
  const R = 6371; // km
  const dLat = (lat2 - lat1) * Math.PI / 180;
  const dLon = (lon2 - lon1) * Math.PI / 180;
  const a = Math.sin(dLat/2) ** 2 + Math.cos(lat1*Math.PI/180) * Math.cos(lat2*Math.PI/180) * Math.sin(dLon/2) ** 2;
  return 2 * R * Math.asin(Math.sqrt(a)); // km
}

Token generation proceeds only if distance <= 0.5 km for normal bookings.

Revenue Split Automation

def split_revenue(total):
    return {
        "hospital": total * 0.80,
        "doctor": total * 0.15,
        "platform": total * 0.05,
    }

Triggered by a FastAPI background task when an appointment status changes to completed.

Revenue Split & System Ethics

  • 80 % → Hospital Manager
  • 15 % → Consulting Doctor
  • 5 % → Platform Service Fee

No VIP bypass – strict FIFO enforced on the doctor dashboard. Repeated patient absences incur a modest ₹50 penalty to discourage no‑shows.
