NOQ

Smart Hospital Queue Management System

A Full-Stack, Role-Based Healthcare Platform

Developed by Sunil T

The Problem

  • Manual Processes: Reliance on paper registers and verbal calls.
  • High Wait Times: Patients wait for hours without visibility into queue position.
  • Operational Blindspots: Hospital managers lack real-time data on hospital efficiency.
  • Revenue Leakage: Manual billing processes lead to discrepancies.
  • Poor Feedback Loop: No centralized system for handling patient complaints or reviews.

The Solution: NOQ Platform

A centralized digital ecosystem unifying 4 key stakeholders:

Super Admin
Global oversight
↔
Hospital Mgr (HM)
Hospital operations
↔
Doctors
Consultations & Queues
↔
Patients
Bookings & Records

Technology Stack

Frontend (UI)

  • React 19 + Vite 7
  • React Router DOM 7
  • Vanilla CSS (Glassmorphism)
  • FontAwesome Icons

Backend & Database

  • Python FastAPI (Async REST)
  • Uvicorn ASGI Server
  • MongoDB (Motor Async Driver)
  • JWT Authentication (PBKDF2)

System Architecture

  • Client-Server Model: Independent React SPA communicating via REST APIs.
  • Modular Design: 18 distinct API route modules and 6 business logic services.
  • Data Layer: MongoDB handles dynamic documents (Queues, Prescriptions, Audits).
  • Stateless Auth: Secure JWT Bearer tokens passed in HTTP headers.

Core Modules

Smart Queuing

Atomic token generation. Doctors can monitor waiting list and "Call Next" in real-time.

Advanced Booking

Priority scheduling with auto doctor allocation based on active load and specialization.

Revenue Analytics

Automated 15/80/5 revenue splits (Doc/HM/Admin) logged instantly on completion.

Clinical Records

Digital prescriptions, refill tracking, and centralized medical history.

Security & Compliance

  • Role-Based Access Control (RBAC): Strict frontend route guards + backend dependency validation.
  • Password Security: PBKDF2-HMAC-SHA256 with 100,000 iterations & random salting.
  • Audit Trail: Every critical action (Create, Update, Delete, Login) is logged centrally with IP tracking.
  • Soft Deletes: Hospitals and Users are never hard-deleted to preserve data integrity.

Advanced Policy & Features

  • Geofenced Booking: Normal tokens must be booked from hospital surroundings. At-home booking support is strictly restricted to Pregnancy, Babies (<7y), and Elders (70+).
  • Zero Emergencies: Excluded entirely from app to focus on OPD queue stability. Critical patients bypass directly.
  • Systemic Equality: Strict FIFO queue sequence with no VIP lane or skip configurations.
  • Accountability: Patients reported 'Absent' get warnings and small penalty fees. Doctor reports trigger manager warnings/suspensions.

Future Scope

  • WebSockets / Server-Sent Events (SSE) for instant live queue updates.
  • Automated SMS and Email notifications for booking confirmations.
  • Self-service Kiosks with QR code check-in at hospitals.
  • Integration with Government Health IDs (ABHA / ABDM).
